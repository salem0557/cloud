# بوت تيليجرام لمسح الأسهم الأمريكية 📊

بوت يمسح **كل الأسهم الأمريكية** (NYSE + Nasdaq + AMEX، حوالي 5000 سهم) على **فريم الساعة** بشكل **متواصل خلال ساعات التداول الرسمية فقط (9:30 صباحاً - 4:00 مساءً بتوقيت نيويورك)** — فور انتهاء دورة المسح تبدأ التالية مباشرة، ويتوقف البوت تماماً خارج هذه الساعات — بحثاً عن **احتمالات الصعود فقط** (استراتيجية طويلة/شرائية بحتة، لا تصعيد ولا عقود Put): يرسل تنبيهاً عند تحقق **فلترين من 3** على الأقل:

1. 📉 السعر عند الحد السفلي لبولينجر باند (20، 2)
2. 🛡️ السعر وصل لمنطقة دعم (قيعان متكررة)
3. 📐 نموذج وتد هابط (Falling Wedge)

البوت **يدفع كل سهم فور اكتشافه** أثناء المسح — لا ينتظر اكتمال الدورة ليجمع النتائج. وهو **لا يكرر النتائج**: يرسل فقط الأسهم الجديدة أو التي تغيّرت فلاترها منذ آخر مسح.

## آلية المسح الذكية

- **قائمة مؤهلة مرتين يومياً**: الأولى **19:30 ET** (قبل افتتاح الجلسة العادية التالية، بما فيها أول يوم بعد الويكند)، والثانية **بعد إغلاق الجلسة العادية (16:30 ET)**؛ بقية الوقت تُفحص القائمة المؤهلة فقط — دورات أسرع وطلبات أقل على Yahoo (الأوقات قابلة للتغيير عبر `QUALIFY_REBUILD_TIMES`)
- **قائمة ساخنة 🔥**: أي رمز يحقق فلترين من أربعة يُراقب على مسار سريع كل دقيقتين، فالإشارة التي تكتمل بين الدورات تصلك خلال دقائق (تنبيهها يبدأ برمز 🔥)
- **تهدئة تلقائية مؤقتة**: إذا بدأ Yahoo برفض الطلبات يضيف البوت مهلاً متزايدة بين الدفعات، وبمجرد التعافي تزول المهل ويعود لسرعته الكاملة تلقائياً
- **توقف تام خارج ساعات التداول**: البوت **يتوقف بالكامل عن المسح** خارج الجلسة الرسمية (9:30 صباحاً - 4:00 مساءً بتوقيت نيويورك، أيام الأسبوع فقط) — يشمل ذلك المساء والليل وما قبل الافتتاح وعطلة نهاية الأسبوع وأيام العطل الرسمية الأمريكية (تُحسب تلقائياً: رأس السنة، MLK، عيد الرؤساء، الجمعة العظيمة، Memorial Day، Juneteenth، عيد الاستقلال، عيد العمال، الشكر، الكريسماس) — ويعاود المسح تلقائياً مع افتتاح الجلسة التالية. يمكن تعطيل إيقاف الويكند/العطل بـ `WEEKEND_HOLIDAY_PAUSE_ENABLED=0`، وتعطيل قصر المسح على ساعات الجلسة الرسمية (والعودة للمسح طوال اليوم كأيام الأسبوع) بـ `MARKET_HOURS_ONLY_ENABLED=0`

## مصدر البيانات

- قائمة الأسهم: ملفات Nasdaq Trader الرسمية (تُحدَّث يومياً، تُستبعد منها صناديق ETF والإصدارات التجريبية)
- الأسعار: Yahoo Finance عبر مكتبة `yfinance` — مجاني وبدون مفاتيح API
- فلتر سيولة تلقائي يستبعد الأسهم تحت 2$ أو قليلة التداول

## التشغيل

### 1. إنشاء البوت
راسل [@BotFather](https://t.me/BotFather) في تيليجرام، أرسل `/newbot`، واحفظ التوكن.

### 2. الإعداد
```bash
git clone <repo-url> && cd cloud
pip install -r requirements.txt
cp .env.example .env    # ثم ضع التوكن داخل .env
```

### 3. التشغيل
```bash
python bot.py
```

ثم افتح البوت في تيليجرام وأرسل `/start`.

 ⚠️ البوت يحتاج سيرفراً يعمل 24/7 (VPS، Railway، Render...). المسح الكامل لـ 5000 سهم يستغرق عادة 15-40 دقيقة حسب سرعة الشبكة وحدود Yahoo — وهو ضمن دورة الساعة. لن يتداخل مسحان أبداً (قفل داخلي).

### النشر على Railway

1. اربط المستودع، وتأكد أن الفرع المنشور هو الفرع الذي يحوي الكود
2. في تبويب **Variables** أضف `TELEGRAM_BOT_TOKEN`
3. **فاليوم دائم (مهم)**: من قائمة الخدمة اختر **Attach Volume** وحدد مسار التركيب مثلاً `/data` — البوت يكتشف الفاليوم تلقائياً (عبر `RAILWAY_VOLUME_MOUNT_PATH`) ويحفظ فيه المشتركين وسجل منع التكرار، فلا تفقد شيئاً عند إعادة النشر ولا تحتاج إرسال `/start` من جديد

على أي استضافة أخرى: عيّن `DATA_DIR=/path/to/persistent/dir` لنفس الغرض.

### الرسم البياني المرفق 📈

كل إشارة تصل كصورة (شموع يابانية على فريم الساعة) مع النص كتعليق عليها، وتضم:

- **حزمتا بولينجر** (الحد العلوي والسفلي والوسط)
- **خط الدعم** (متقطع أخضر) عند المستوى الذي رصده الفلتر، إن وُجد
- **لوحة RSI فرعية** أسفل الرسم مع خط متقطع أحمر عند مستوى التشبع البيعي (30) — للسياق فقط، لم يعد RSI أحد شروط التنبيه
- عدد الشموع المعروضة قابل للتغيير عبر `CHART_BARS` (الافتراضي 80)، والتعطيل الكامل عبر `CHART_ENABLED=0`

إن كان نص الإشارة طويلاً جداً (تعليق الصورة في تيليجرام محدود بـ1024 حرفاً) يصل الرسم بتعليق مختصر متبوعاً برسالة نصية منفصلة بكامل التفاصيل — تلقائياً وبدون أي إعداد إضافي.

### عقود الأوبشنز المرفقة 📊

مع كل إشارة سهم يجلب البوت سلسلة الأوبشنز ويرفق **أفضل 3 عقود Call 🟢📈 فقط** (البوت استراتيجية صعود بحتة — لا عقود Put إطلاقاً) — **ليس الأرخص بالضرورة**، بل الأعلى دلتا وفق شرط واحد بسيط:

- **النطاق الزمني**: من أقرب انتهاء حتى ~3 أشهر تقريباً (`OPTIONS_MAX_WEEKS`، افتراضياً 13 أسبوعاً)
- **شرط الاختيار الوحيد**: دلتا أكثر من 0.40 (`DELTA_MIN`) — العقود المؤهلة تُرتَّب من الأعلى دلتا إلى الأقل، وتُستبعد العقود الميتة (بلا نشاط) أو بلا عرض شراء حي بغض النظر عن دلتاها
  - الدلتا غير متوفرة مباشرة من Yahoo أو CBOE، فتُحسب تقديرياً عبر معادلة Black-Scholes من السعر الحالي وسعر التنفيذ والأيام المتبقية والتقلب الضمني
- **لا عقود مناسبة = لا تنبيه إطلاقاً**: أي سهم يحقق فلاتر الاتجاه الفني لكن لا يوجد له ولا عقد Call صالح **لا يُرسل تنبيهه أبداً** — لا يُعتبر إشارة كاملة بدون خطة تداول عملية معه. السهم لا يُسجَّل في ذاكرة التكرار في هذه الحالة، فيُعاد فحصه في كل دورة مسح لاحقة تلقائياً (قد تتحسن سيولة عقوده لاحقاً فيظهر)
- **مصدران للبيانات**: Yahoo أولاً، وعند فشله ينتقل تلقائياً إلى بورصة CBOE (سلسلة كاملة بطلب واحد، مجاناً وبدون مفاتيح) — ولا يُعتبر السهم «بلا عقود» إلا بتأكيد المصدرين معاً؛ وإن كانت كل عقوده أبعد من النطاق الزمني تظهر رسالة مختلفة توضح ذلك بدل الادعاء بعدم وجود أوبشن أصلاً
- يظهر لكل عقد: سعر التنفيذ، تاريخ الانتهاء (وعدد الأيام)، البريميوم للسهم وللعقد الكامل (×100)، ونسبة IV والدلتا والثيتا إن أمكن حسابها
- التعطيل: `OPTIONS_ENABLED=0`

### سجل الأداء الفعلي 📊

كل إشارة جديدة تُسجَّل تلقائياً بسعرها اللحظي وسعر مؤشر **SPY** في نفس اللحظة (حساب رقمي بحت، بلا أي ذكاء اصطناعي أو تكلفة إضافية — بيانات ياهو المجانية فقط). بعد **24 ساعة و72 ساعة** (قابلتان للتغيير عبر `PERFORMANCE_HORIZONS_HOURS`) يتحقق البوت تلقائياً من عائد السهم مقارنة بعائد SPY لنفس الفترة (الفارق = "ألفا")، ويبني سجلاً حقيقياً لأداء فلاترك بمرور الوقت. إشارة الصعود تفوز إن تفوق السهم على SPY (ألفا موجبة).

بمجرد توفر **5 نتائج مؤكدة** على الأقل لأي مدة (لتفادي رقم مضلل من عيّنة صغيرة)، يُرفق سطر مختصر تلقائياً **أسفل كل تنبيه جديد**:
```
📊 سجل الأداء: تفوق على السوق في 61% من آخر 42 إشارة (بعد 24 ساعة) — التفاصيل: /performance
```

### وجهة نظر البوت 💬

تحت كل إشارة (إن توفرت بيانات) يظهر ملخص قصير يجمع **آخر عناوين الأخبار من ياهو** و**تعليقات المتداولين على StockTwits** عن السهم، ويدمجها بفقرة عربية واحدة عبر استدعاء واحد رخيص لنموذج **Google Gemini** (المجاني). القسم **تلخيص ودمج بحت** — لا يُبدي رأياً أو توصية مستقلة، فقط يذكر الاتجاه العام الذي تعكسه المصادر نفسها (هذا مكتوب صراحة في الأمر الموجَّه للنموذج).

**التفعيل** (اختياري ومجاني بالكامل):
1. افتح [aistudio.google.com](https://aistudio.google.com) وسجّل دخول بحساب Google
2. اضغط **Get API key** وأنشئ مفتاحاً (بلا بطاقة ائتمان)
3. أضفه في Railway → Variables باسم `GEMINI_API_KEY`

بدون هذا المفتاح يُعطَّل القسم تلقائياً بلا أي خطأ أو تأثير على بقية التنبيه. التعطيل اليدوي: `SENTIMENT_ENABLED=0`.

`/performance` (متاح للجميع، بلا حاجة لاشتراك) يعرض:
```
📊 سجل أداء الإشارات (السهم مقابل مؤشر SPY):
  بعد 24 ساعة: 42 إشارة • تفوق على السوق في 61% منها • متوسط الفارق +1.35%
  بعد 72 ساعة: 38 إشارة • تفوق على السوق في 58% منها • متوسط الفارق +2.10%
⏳ 6 إشارات قيد المتابعة (لم يحن وقت تقييمها بعد)
```
التعطيل: `PERFORMANCE_ENABLED=0`

### فحص سهم عند الطلب 🔍

`/check <رمز>` — مثال: `/check AAPL` — يجلب السهم فوراً بنفس إضافات التنبيه العادي (رسم بياني، عقود أوبشنز، وجهة نظر البوت)، **بغض النظر عن نتيجة فلاتره الفعلية حالياً**. الرد يصلك أنت فقط (لا يُبث لبقية المشتركين)، ولا يُسجَّل في سجل الأداء لأنه ليس إشارة فعلية.

إذا أرسلت `/check` بلا رمز (شائع عند الضغط على اقتراح الأمر من قائمة تيليجرام، والتي ترسله فوراً دون فرصة لكتابة أي شيء بجانبه) يسألك البوت عن الرمز في رسالة منفصلة بدل إجبارك على إعادة كتابة الأمر كاملاً — فقط أرسل الرمز في الرسالة التالية خلال 3 دقائق.

### بحث عن عقود أوبشنز رخيصة 💰

`/cheapoptions [الحد الأقصى لسعر العقد]` — مثال: `/cheapoptions 50` (الافتراضي 50$ إن لم تحدد رقماً) — يمسح **القائمة المؤهلة الحالية** (نفس الأسهم السائلة التي تُفحص فنياً بشكل دوري) بحثاً عن أي سهم لديه عقد Call **موثوق** (يجتاز نفس معايير السيولة وصحة السعر المستخدمة في كل تنبيه) بسعر شراء لا يتجاوز المبلغ المحدد — سعر العقد الكامل = البريميوم × 100 سهم، فـ 50$ تعني بريميوم لا يتجاوز 0.50$ للسهم الواحد.

- الرد يصلك أنت فقط، ولا علاقة له بالتنبيهات التلقائية أو سجل الأداء
- إن لم تكن القائمة المؤهلة قد بُنيت بعد يطلب منك تشغيل `/scan` أولاً
- **قد يستغرق عدة دقائق**: لا يوجد API واحد يعطي أسعار أوبشنز لعدة أسهم دفعة واحدة، فالبوت يفحص كل سهم في القائمة على حدة (رسالة تقدّم دورية أثناء الانتظار)؛ لا يمكن تشغيل بحثين في آن واحد (قفل داخلي)
- التخصيص: `CHEAP_OPTION_DEFAULT_MAX` (القيمة الافتراضية)، `CHEAP_OPTIONS_PACE_SECONDS` (مهلة بين كل سهم وآخر لتفادي الحظر)

| `MIN_AVG_VOLUME` | 30000 | الحد الأدنى لمتوسط حجم التداول للشمعة |

## الاشتراك المدفوع وإخلاء المسؤولية

البوت مجهّز للعمل كخدمة اشتراك مدفوع مع بوابة قانونية:

- **إخلاء مسؤولية إلزامي**: عند /start يظهر إخلاء مسؤولية كامل (الخدمة تحليل فني آلي، ليست توصية، غير مرخصة من هيئة السوق المالية) ولا يستقبل المستخدم أي إشارة قبل الضغط على زر الموافقة — والموافقة مسجلة بوقتها في حالة البوت
- **تذييل كل تنبيه**: «⚠️ تحليل فني آلي — ليس توصية بشراء أو بيع»
- **التفعيل اليدوي**: المستخدم يوافق ثم يظهر له معرّفه ووسيلة التواصل (`SUBSCRIBE_CONTACT`)؛ بعد الدفع تفعّله بأمر `/approve <id> <أيام>` (0 = دائم)، وينتهي تلقائياً بإشعار للمستخدم
- **أوامر المشرف** (`ADMIN_CHAT_ID`): `/approve` تفعيل • `/revoke` إلغاء • `/subs` قائمة المشتركين
- `/disclaimer` يعرض الإخلاء كاملاً في أي وقت
- `/previewalert [رمز]` (مشرف فقط): يرسل لكل المشتركين الحاليين مثالاً حقيقي البيانات لشكل التنبيه الجديد، مع بانر بارز في أعلى الرسالة يوضح أنه **مثال توضيحي وليس إشارة حقيقية** — مفيد عند إضافة ميزة جديدة للتنبيه لإطلاع المشتركين على الشكل الجديد. الافتراضي `AAPL` إن لم تحدد رمزاً. لا يُسجَّل في سجل الأداء (`/performance`) لأنه ليس إشارة فعلية.

> ⚠️ هذا الإخلاء صيغ بعناية لكنه ليس استشارة قانونية — إن كنت ستبيع اشتراكات من داخل السعودية فاعرض الصيغة على مختص قانوني، فتقديم «توصيات» مالية مقابل مقابل مادي نشاط منظم من هيئة السوق المالية حتى مع وجود إخلاء مسؤولية.

اكتشاف الأنماط (الدعم والوتد الهابط) يعتمد على خوارزميات تقريبية وقد يختلف عن التحليل اليدوي.

# US Options Screener

A command-line agent that scans US-listed stock options and surfaces the
contracts that best match a standard set of liquidity/pricing criteria:

- **Bid** / **Ask** (and the bid-ask spread)
- **Volume**
- **Open Interest**
- **Implied Volatility (IV)**
- **Delta**
- **Theta**

Data comes from Yahoo Finance (free, unofficial, delayed ~15-20 min).
Delta and Theta are **not** provided by Yahoo, so they are computed locally
with the Black-Scholes model from each contract's implied volatility, the
underlying's spot price, strike, and time to expiry.

## Install

```bash
pip install -r requirements.txt
```

## Usage

```bash
# Scan the S&P 500 (default universe), show top 25 by volume
python3 main.py

# Scan specific tickers, calls only, wider expiry window
python3 main.py --universe AAPL,MSFT,TSLA,NVDA --option-type call --min-dte 14 --max-dte 60

# Scan the entire US market (thousands of tickers - can take a long time
# against a free, rate-limited data source; consider --max-tickers for a
# quick test first)
python3 main.py --universe all --max-tickers 200 -v
```

`--universe` accepts:
- `sp500` (default) - S&P 500 constituents, scraped from Wikipedia
- `all` - every stock/ETF listed on Nasdaq/NYSE/NYSE American
- a comma-separated list of tickers, e.g. `AAPL,MSFT`
- a path to a text file with one ticker per line

## Default filter thresholds

These are common, configurable screening conventions - not investment
advice - meant to surface liquid, reasonably priced, near-the-money
contracts a few weeks to two months out:

| Filter | Default | Flag |
|---|---|---|
| Days to expiry | 7-45 | `--min-dte` / `--max-dte` |
| Min volume | 100 | `--min-volume` |
| Min open interest | 500 | `--min-open-interest` |
| Max bid/ask spread | 10% of mid price | `--max-spread-pct` |
| IV range | 15%-100% | `--iv-min` / `--iv-max` |
| \|Delta\| range | 0.30-0.70 | `--delta-min` / `--delta-max` |
| Max theta burn | 5% of mid price/day | `--max-theta-pct` (pass `-1` to disable) |

Run `python3 main.py --help` for the full list of options, including
`--sort-by`, `--max-workers`, and `--request-delay` (throttling to avoid
getting rate-limited by Yahoo).

## Tests

```bash
python3 -m unittest discover -v tests
```

Tests cover the Black-Scholes Greeks math and the filter logic; they don't
require network access. Actually scanning the market does require outbound
HTTPS access to `query2.finance.yahoo.com` (and, for `--universe sp500` /
`all`, to Wikipedia / nasdaqtrader.com) - some sandboxed environments block
this, in which case the CLI runs but reports zero results.
