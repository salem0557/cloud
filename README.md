# بوت تيليجرام لمسح الأسهم والأوبشن والكريبتو 📊

بوت بثلاث وحدات **مستقلة تماماً** عن بعضها، تعمل فقط عند الطلب اليدوي (لا مسح تلقائي مستمر في الخلفية):

- 📈 **الأسهم** (`/stocks`) — إشارات ارتداد صعودي فنية
- 📊 **الأوبشن** (`/options`) — عقود CALL مؤهلة وفق فلاتر سيولة/دلتا/سعر صارمة
- 🪙 **الكريبتو** (`/crypto`) — نفس منطق الأسهم على أعلى 30 عملة بالقيمة السوقية

كل وحدة لها **قائمة مراقبة خاصة بها**، **فلاتر خاصة بها**، ومعالجة أخطاء مستقلة تماماً — فشل وحدة واحدة (مثلاً انقطاع مزوّد بيانات) لا يوقف الوحدتين الأخريين إطلاقاً.

## آلية الجلسات

كل أمر (`/stocks` أو `/options` أو `/crypto`) يبدأ **جلسة فحص واحدة لكل محادثة**:

- تتوقف تلقائياً بعد 5 دقائق كحد أقصى (`SESSION_TIMEOUT_SECONDS`) وترسل رسالة "⏰ انتهت الجلسة"
- `/stop` يوقفها فوراً في أي لحظة
- لا يمكن بدء جلسة جديدة أثناء تنفيذ جلسة أخرى لنفس المحادثة

## الوصول: مقفل على الأعضاء الحاليين فقط

البوت **لا يحتوي أي أمر لإضافة أعضاء جدد** (لا `/approve` ولا `/start`). الوصول مقتصر على قائمة ثابتة من معرّفات المحادثات المعتمدة مسبقاً، محفوظة في ملف الحالة (`state.json`، الحقل `approved`). لإضافة أو إزالة عضو الآن يجب تعديل هذا الملف يدوياً على السيرفر — لا توجد طريقة من داخل البوت.

كل نتيجة تُرسل مذيّلة بإخلاء مسؤولية مختصر (تحليل فني آلي، ليس توصية استثمارية).

## الوحدة 1: الأسهم (`/stocks`)

تفحص **STOCKS_WATCHLIST** (حوالي 290 سهماً من Nasdaq-100 وأكبر أسهم S&P 500 عبر كل القطاعات — قائمة ثابتة في `scanner/config.py`، وليست متزامنة تلقائياً مع مؤشر حي) على فريم يومي، وتتطلب تحقق **3 من 4** فلاتر:

1. 📉 السعر عند الحد السفلي لبولينجر باند (ضمن 2%)
2. 🔻 RSI أقل من 35 (فترة 14)
3. 🛡️ السعر عند منطقة دعم (قيعان متكررة خلال آخر 60 يوماً، ضمن هامش 3%)
4. 📐 نموذج وتد هابط (Falling Wedge)

النتيجة: أفضل 5 أسهم، كل نتيجة بسطرين، مع **نسبة الربح المحتملة** بارزة في نهاية السطر الثاني (المسافة إلى أقرب مقاومة تاريخية، أو 10%+ افتراضياً إن لم توجد مقاومة واضحة).

## الوحدة 2: الأوبشن (`/options`, `/options <رمز>`)

تفحص **OPTIONS_WATCHLIST** (قائمة منفصلة تماماً — حوالي 100 من أكثر الأسهم نشاطاً بعقود الأوبشن، لا علاقة لها بإشارات وحدة الأسهم) بحثاً عن عقود CALL تحقق **كل** الشروط التالية دفعة واحدة:

- دلتا بين 0.55 و0.80 (Black-Scholes يدوي بدون مكتبات خارجية — انظر ملاحظة أدناه)
- أيام حتى الانتهاء (DTE) بين 45 و120
- حجم تداول ≥ 30 وعقود مفتوحة ≥ 200
- تقلب ضمني (IV) أقل من 60%
- سبريد عرض/طلب أقل من 10%
- سعر الطلب (ask) بين 0.30$ و1.50$ للسهم (تكلفة العقد = ask × 100)

`/options` بلا رمز يفحص القائمة كاملة. `/options AAPL` يفحص سهماً واحداً فقط بمعزل عن القائمة. النتيجة: أفضل 5 عقود بأعلى **احتمالية ربح (Probability of Profit)** — وهي "نسبة الربح المحتملة" البارزة في نهاية كل نتيجة، محسوبة من التوزيع اللوغاريتمي الطبيعي لسعر السهم عند الانتهاء مقابل نقطة تعادل العقد.

**لماذا Black-Scholes يدوي بدل py_vollib/mibian؟** نشر سابق لهذا البوت تعطّل لأسابيع بسبب سطرين غريبين تسللا لملف `requirements.txt` بعد دمج (merge) لم يُلاحظ، فمنع `pip install` من النجاح على كل نشر لاحق. لتقليل الاعتماد على حزم خارجية جديدة قد تفشل بصمت، الدلتا/الثيتا تُحسب بدالة `math.erf` القياسية في بضعة أسطر بايثون خالصة (`scanner/options.py`) بدل إضافة مكتبة تسعير خارجية.

## الوحدة 3: الكريبتو (`/crypto`)

تفحص **CRYPTO_WATCHLIST** (أعلى 30 عملة بالقيمة السوقية، أزواج USDT على Binance) على فريم **4 ساعات** عبر بيانات Binance العامة (`ccxt`، بلا مفاتيح API)، وتتطلب تحقق **2 من 3** فلاتر (بلا وتد هابط، خاص بالأسهم فقط):

1. 📉 بولينجر السفلي (ضمن 2%)
2. 🔻 RSI أقل من 35
3. 🛡️ منطقة دعم (آخر 30 يوماً = 180 شمعة 4 ساعات، ضمن هامش 3%)

نفس منطق "نسبة الربح المحتملة" (المسافة إلى أقرب مقاومة) المستخدم في وحدة الأسهم.

## أوامر البوت

| الأمر | الوظيفة |
|---|---|
| `/stocks` | فحص وحدة الأسهم (أفضل 5 نتائج) |
| `/options` | فحص وحدة الأوبشن كاملة (أفضل 5 عقود) |
| `/options <رمز>` | فحص عقود سهم واحد فقط |
| `/crypto` | فحص وحدة الكريبتو (أفضل 5 نتائج) |
| `/stop` | إيقاف الجلسة الحالية فوراً |
| `/status` | حالة عضويتك وجلستك الحالية والسوق |

لا توجد أوامر أخرى — لا اشتراك مدفوع، لا إدارة أعضاء، لا مسح تلقائي، لا رسوم بيانية.

## مصادر البيانات

- الأسهم/الأوبشن: Yahoo Finance عبر `yfinance` (احتياطي لسلاسل الأوبشن عبر CBOE عند تعطل Yahoo)
- الكريبتو: Binance العامة عبر `ccxt` (بيانات سوق فقط، بلا مفاتيح API)

## التشغيل

### 1. إنشاء البوت
راسل [@BotFather](https://t.me/BotFather) في تيليجرام، أرسل `/newbot`، واحفظ التوكن.

### 2. الإعداد
```bash
git clone <repo-url> && cd cloud
pip install -r requirements.txt
cp .env.example .env    # ثم ضع التوكن داخل .env
```

### 3. اعتماد الأعضاء
أضف معرّفات المحادثات المسموح لها يدوياً في ملف الحالة (`state.json` داخل `DATA_DIR`، الافتراضي المجلد الحالي):
```json
{"approved": {"123456789": 0}}
```
`0` = اشتراك دائم، أو رقم Unix timestamp لتاريخ الانتهاء.

### 4. التشغيل
```bash
python bot.py
```

⚠️ البوت لا يحتاج تشغيلاً متواصلاً 24/7 بالضرورة (لا مسح خلفي) لكن يُفضّل تشغيله على سيرفر دائم (VPS، Railway، Render...) ليستجيب للأوامر في أي وقت.

### النشر على Railway

1. اربط المستودع، وتأكد أن الفرع المنشور هو الفرع الذي يحوي الكود
2. في تبويب **Variables** أضف `TELEGRAM_BOT_TOKEN`
3. **فاليوم دائم (مهم)**: من قائمة الخدمة اختر **Attach Volume** وحدد مسار التركيب مثلاً `/data` — البوت يكتشف الفاليوم تلقائياً (عبر `RAILWAY_VOLUME_MOUNT_PATH`) ويحفظ فيه ملف الأعضاء المعتمدين

على أي استضافة أخرى: عيّن `DATA_DIR=/path/to/persistent/dir` لنفس الغرض.

## متغيرات البيئة الرئيسية

| المتغيّر | الافتراضي | الوصف |
|---|---|---|
| `TELEGRAM_BOT_TOKEN` | — | توكن البوت (مطلوب) |
| `ADMIN_CHAT_ID` | 0 | معرّفك الشخصي (مصرّح له دائماً بلا حاجة لإدراجه في `approved`) |
| `SUBSCRIBE_CONTACT` | "مشغّل البوت" | يظهر في رسالة الرفض للأعضاء غير المعتمدين |
| `SESSION_TIMEOUT_SECONDS` | 300 | الحد الأقصى لطول أي جلسة فحص |
| `STOCKS_FILTERS_REQUIRED` | 3 | من أصل 4 فلاتر أسهم |
| `CRYPTO_FILTERS_REQUIRED` | 2 | من أصل 3 فلاتر كريبتو |
| `OPTIONS_DELTA_MIN` / `OPTIONS_DELTA_MAX` | 0.55 / 0.80 | نطاق الدلتا المقبول |
| `OPTIONS_ASK_MIN` / `OPTIONS_ASK_MAX` | 0.30 / 1.50 | نطاق سعر الطلب للسهم الواحد |
| `DATA_DIR` | . | مجلد حفظ `state.json` |

القائمة الكاملة للمتغيرات (فلاتر بولينجر/RSI/الدعم/الوتد لكل وحدة، حدود سيولة عقود الأوبشن، إلخ) موجودة كتعليقات في `scanner/config.py`.

## هيكل المشروع

```
bot.py                    # الأوامر الستة + آلية الجلسات (5 دقائق + /stop)
scanner/
  config.py                # فلاتر ثلاث الوحدات + القوائم الثلاث + إعدادات عامة
  state.py                 # قائمة الأعضاء المعتمدين الثابتة (state.json)
  indicators.py             # بولينجر/RSI/الدعم/المقاومة/الوتد الهابط (معاملات صريحة)
  data.py                   # تنزيل OHLCV دفعي عبر yfinance (أسهم/أوبشن)
  crypto_data.py            # OHLCV عبر ccxt/Binance العامة (كريبتو)
  options.py                # جلب سلاسل عقود Call (Yahoo + CBOE) + دلتا/ثيتا يدوية
  pricing.py                 # تسعير Black-Scholes نظري، نقطة تعادل، أقصى خسارة
  probability.py             # احتمالية الربح (Probability of Profit)
  market_calendar.py         # هل السوق الأمريكي مفتوح الآن (لعرض /status فقط)
  stocks_module.py           # وحدة الأسهم (مستقلة)
  options_module.py          # وحدة الأوبشن (مستقلة)
  crypto_module.py           # وحدة الكريبتو (مستقلة)
```
